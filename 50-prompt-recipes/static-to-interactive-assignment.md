# Prompt Recipe: Static to Interactive Assignment

## Goal
Convert a traditional assignment into more interactive, scaffolded, or guided activity.

## Use this when
- you have a reading response, worksheet, or essay prompt that feels too static
- you want more check-ins, practice, or guided progression before a final submission

## Gather these materials
- the current assignment prompt
- learning objectives
- grading criteria or rubric if available
- module materials or readings
- student level and delivery format
- accessibility, Canvas, or hosting constraints

## Copy/paste prompt
```text
You are helping me redesign a traditional assignment into a more interactive learning activity.

Use the attached materials as the primary source of truth:
- the current assignment prompt
- learning objectives
- rubric or grading criteria, if available
- module readings or course materials
- any student-level, accessibility, Canvas, or hosting constraints I provide

Task:
Redesign this assignment so students get more structure, interaction, and feedback opportunities while still meeting the original course goals.

Important constraints:
- Preserve the core learning objective and assessment purpose of the original assignment.
- Do not add tools, media, or technical requirements unless they fit the supplied constraints.
- Avoid adding interaction that is only cosmetic or busywork.
- If a redesign depends on missing platform or implementation details, mark that as [NEEDS REVIEW].
- Keep the activity realistic for the course level, time available, and instructional setting.

Return:
1. short summary of the original assignment purpose
2. redesigned interactive version
3. step-by-step student flow
4. note explaining how the redesign preserves the original learning goals
5. any [NEEDS REVIEW] implementation questions
```

## Watchouts
- Do not convert a strong assignment into a flashy but weaker activity.
- Do not assume interactivity requires custom code; structure and scaffolding often matter more.
- Do not remove rigor when adding guidance.
- Do not ignore accessibility or Canvas limitations when proposing formats.

## Revision moves
- add the current rubric if the redesign drifts away from what is actually assessed
- ask for a lower-tech version if hosting or JavaScript is unrealistic
- ask for a Canvas-native version if the output needs to live inside a module
- ask for a lighter-weight redesign if the first pass adds too many steps
