# Institutional Requirements

## Purpose
Prompts should embed required institutional or departmental elements rather than leaving them implicit.

## Use this when
- revising a syllabus
- drafting assignment language that must reflect program rules
- checking whether required accessibility, integrity, or attendance language is present

## Best prompt pattern
- attach the actual requirement language
- tell the model which document is the source of truth
- ask it to identify missing, outdated, or uncertain items
- prohibit invention of policies not present in the supplied materials

## Watchouts
- do not ask the model to "add the normal university policies" without providing them
- do not treat an older syllabus as authoritative if the current policy source is missing
- do not let the model silently merge policy language from multiple eras without review
