# Rutgers Tooling

## Approved AI tools

Rutgers faculty have access to two approved AI tools. For current guidance, approved use cases, and policy, start at **[it.rutgers.edu/ai](https://it.rutgers.edu/ai)**.

### Google Gemini
Available to Rutgers faculty and staff via ScarletMail (Google Workspace).
→ [gemini.google.com](https://gemini.google.com/)

### Microsoft 365 Copilot
Available to Rutgers faculty and staff via Microsoft 365.
→ [copilot.microsoft.com](https://copilot.microsoft.com/)

Note: Rutgers provides Microsoft 365 Copilot, not the full standalone Copilot product. Feature availability may differ from what is described in general Microsoft documentation.

## Observed limitations

- Inconsistent file export behavior across tools
- Partial Office integrations — not fully embedded in all M365 apps
- Differences in multimodal capabilities (image and document understanding varies)
- Some "agent-like" features are not true agents — they do not reliably chain multi-step tasks

## Canvas

Canvas is the Rutgers LMS. AI-generated materials that need to be delivered to students must be hosted — either in Canvas directly or externally with an iframe or link.
→ [canvas.rutgers.edu](https://canvas.rutgers.edu/)

## Policy and acceptable use

All AI tool use at Rutgers should follow current IT guidance.
→ [it.rutgers.edu/ai](https://it.rutgers.edu/ai)
